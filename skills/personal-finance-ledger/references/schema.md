# Schema Reference

Read this file when creating, validating, reading, or writing transaction records.

## Google Sheet

Default spreadsheet ID:

```text
FINANCE_SHEET_ID
```

Default sheet name:

```text
Transactions
```

## Transactions header

The first row must contain these headers in this order:

```text
transaction_id
transaction_date
type
amount
currency
category
description
merchant
payment_method
source
receipt_file_ref
ocr_confidence
notes
created_at
updated_at
```

## Field definitions

### `transaction_id`

Unique ID generated by the runtime.

Recommended format:

```text
txn_<YYYYMMDD>_<random_or_hash>
```

Example:

```text
txn_20260531_a8f32c
```

### `transaction_date`

Date of transaction.

Format:

```text
YYYY-MM-DD
```

### `type`

Allowed values:

```text
income
expense
```

### `amount`

Positive number only.

Do not store negative values.

Use `type` to distinguish income from expense.

### `currency`

ISO-like currency code.

Default:

```text
IDR
```

### `category`

Human-readable category.

Examples:

```text
Makanan & Minuman
Transportasi
Gaji
Belanja
Lainnya
```

### `description`

Short normalized description.

Example:

```text
beli kopi
```

### `merchant`

Merchant, store, employer, or payer name if known.

Examples:

```text
Indomaret
Starbucks
PT Contoh Sejahtera
```

Can be empty if unknown.

### `payment_method`

Payment method if known.

Examples:

```text
cash
debit
credit_card
qris
bank_transfer
e_wallet
```

Can be empty if unknown.

### `source`

Allowed values:

```text
text
receipt
manual
import
correction
```

### `receipt_file_ref`

Reference to uploaded receipt file if available.

Do not store raw image data in the sheet.

### `ocr_confidence`

Number from `0` to `1`.

Use empty value if OCR was not used.

### `notes`

Optional notes.

### `created_at`

Record creation timestamp.

Format:

```text
YYYY-MM-DDTHH:mm:ssZ
```

or timezone-aware ISO 8601.

### `updated_at`

Last update timestamp.

Empty for newly appended records unless the runtime always fills it.

## Example row

```csv
transaction_id,transaction_date,type,amount,currency,category,description,merchant,payment_method,source,receipt_file_ref,ocr_confidence,notes,created_at,updated_at
txn_20260531_a8f32c,2026-05-31,expense,15000,IDR,Makanan & Minuman,beli kopi,,qris,text,,,2026-05-31T09:15:00+07:00,
```

## Summary calculation

For a filtered date range:

```text
total_income = sum(amount where type = income)
total_expense = sum(amount where type = expense)
balance = total_income - total_expense
```

Category summary:

```text
category_total = sum(amount where type = expense grouped by category)
```

## Validation rules

Before writing:

- `transaction_id` must not be empty.
- `transaction_date` must be valid.
- `type` must be `income` or `expense`.
- `amount` must be greater than `0`.
- `currency` must not be empty.
- `category` must not be empty.
- `description` must not be empty.
- `source` must be valid.
- `created_at` must not be empty.

Do not write invalid rows.
